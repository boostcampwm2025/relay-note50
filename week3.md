
# 토의 내용 정리

## 1번 **🧠 “AI야, 최신 AI 뉴스 줘!”**

### 문제점 / 개선할 점

1. GPT가 최신 정보를 가져올 수 있을까?
    
    ⇒ 가능 : 최근 포스팅이나 글들을 읽어올 수 있다.
    
2. AI에 관심 있는 사람들만 본다는 문제
    
    AI에 큰 관심이 없는 사람들의 슬랙 참여를 유도하기 어렵다.
    

### 해결방안

1. 많은 사람들에 관심이 있을만한 분야로 수정하자.
    - AI가 아닌 자신의 분야별 언어(JS, ios, Kotlin)에 관련된 최신 뉴스를 공유하면 더 좋을 것 같다.

## 2번 **🧸 “내 캐릭터가 슬랙 질문에 답변한다면?”**

### 문제점

1. 캐릭터로 내용을 생성하여 슬랙에 공유하면 쑥스러울 수 있어서 미션을 기피하게 될 수 있다.

### 해결방안

1. 캐릭터보다는, 등장인물 전체로 범위를 넓히자.
    
     → 좀 더 범위를 넓히자.
    

## 3번 **🧑‍🎨 “이 코드, 인격이 있다면 어떤 말을 할까?”**

### 문제점 / 개선할 점 토론

1. 추상적으로 적혀있어서 구체적으로 적어서 쉽게 이해하게 만들면 좋을 것 같다. 
    - 어떤 코드에 인격을 부여한다는 거지?
2. 어떤 함수/코드인지 공감을 유도하기 어렵다.

### 해결방안

1. 코드가 아닌, 함수, 메서드, 클래스 단위로 구체적인 대상으로 인격을 부여하자
    - 자주 사용하는 함수 ex. console.log 와 같은 것
2. 과제 개발할 때 자주 사용하는 함수들을 대상으로 하자.

### 이렇게 개선하면 좋을 것 같아요

코드의 역할을 맞추는 게임 : 과제하면서 자주 사용한 함수를 정해서 해당 역할을 AI에게 정리해달라고 한다. 이때 “해당 코드/함수에 인격을 부여해서 “널 소개해줘”’와 같이 정리해달라고 한다. 

하면서 정리한 내용을 바탕으로 슬랙에 올린다. ⇒ “나는 누굴까?" 문제 맞추기!

## 퀘스트 개선 및 이유

1. 1. 🧠 “AI야, 최신 AI 뉴스 줘!” -> "AI야, 개발분야별 언어에 대한 최신 뉴스 줘 "
    - AI 분야는 관심 있는 사람도 있고, 없는 사람도 있기 때문에 반응 유도 즉 커뮤니티 참여 유도를 이끌어내기 어렵다.
    - JS/코틀린/ios 분야의 최신 뉴스를 슬랙에 공유하면 관심있어하는 사람이 있기 때문에 커뮤니티 참여 유도를 더 적극적으로 이끌어낼수 있을 것 같다.
   
3. 🧸 “내 캐릭터가 내 코드를 본다면?” -> 🧸 “작품 속의 등장인물이 슬랙 질문에 답변한다면?”
   - 캐릭터라고 표현하면 만화나 애니메이션으로 생각하는 경우가 많고, 그럴 경우 다소 민망하고 부끄러움을 느끼는 분들이 계실 수 있다고 생각했다.
   - 그래서 캐릭터보다는, 작품 속의 등장인물로 범위를 넓히자.
     
4. 🧑‍🎨 “이 코드, 인격이 있다면 어떤 모습일까?” -> 🧑‍🎨 “이 코드, 인격이 있다면 어떤 말을 할까?”
   - 이미지 생성은 무겁고 부담스럽다. 미션하기 바쁜데 생성까지 기다리는 시간 조차 용납 못한다!
   - "모습"이 애매모호하다 느껴 좀 더 구체적인 결과("말")를 원했습니다!
     
4. 🎬 “영화 제목이나 명대사로 커밋하기: AI 작명소”
   - 기존 아이디어가 너무 좋아, 수정없이 유지하기로 했습니다.
